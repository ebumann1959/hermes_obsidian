## Show Runner Pi ↔ Unity: Problem Diagnosis

### Status: Unity is not running (immediate blocker)
TCP port 8080 on 10.0.0.91 returns ECONNREFUSED. Unity is off or the ShowRunner scene isn't open with `UnityComms.cs` active. Nothing works until that's running.

---

### Bug #1 — HTTPS vs HTTP protocol mismatch (breaks when Unity starts)

**`pi_unity_client.py:22`:**
```python
BASE_URL = f"https://{PC_ADDR}:{UNITY_PORT}"   # ← HTTPS
```

**`UnityComms.cs` (Start):**
```csharp
_httpListener.Prefixes.Add($"http://+:{port}/api/");  // ← plain HTTP
```

Unity's `HttpListener` speaks HTTP only — no TLS. The Pi is connecting with HTTPS. Every request will fail with a connection error or garbled response. The `verify=False` suppresses cert validation but doesn't fix the underlying protocol mismatch.

**Fix:** Change `pi_unity_client.py:22` to:
```python
BASE_URL = f"http://{PC_ADDR}:{UNITY_PORT}"
```

---

### Bug #2 — Authorization header is never extracted (all mutating endpoints return 401)

In `UnityComms.cs`, `ProcessRequest` receives only `(method, path, body, clientIP)`. The auth header extraction call is:
```csharp
string authToken = ExtractHeader(body, "Authorization");
```

`ExtractHeader` parses header-format text out of a string — but `body` is the **JSON request body**, not the raw HTTP request. The `Authorization` header lives in `context.Request.Headers`, which is never forwarded to `ProcessRequest`. This always returns `null` → every `POST /api/state`, `/api/scene`, `/api/capture` returns 401, even with the correct token.

**Fix:** Extract the header in `HttpListenerLoop` where `context` is available, store it in `PendingRequest`, and pass it through:

```csharp
// In PendingRequest class, add:
public string AuthorizationHeader;

// In HttpListenerLoop, alongside reading body:
string authHeader = context.Request.Headers["Authorization"] ?? "";

var req = new PendingRequest {
    ...
    AuthorizationHeader = authHeader,
    ...
};
```

Then in `ProcessRequest`, replace the `ExtractHeader` call:
```csharp
// Change signature to accept authToken directly, or add it to PendingRequest
string authToken = req.AuthorizationHeader;  // instead of ExtractHeader(body, ...)
```
