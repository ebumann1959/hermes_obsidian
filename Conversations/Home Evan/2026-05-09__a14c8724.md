## Assistant

You can't do it with a single config file — each running Conky process reads one config and targets one `xinerama_head`. The standard approach is exactly what you'd be doing: two config files, one per monitor.

However, you can minimize duplication by using a **shared template approach**:

- One base file with all the `conky.text` content
- Two minimal config files that `dofile()` or symlink the base, each only differing in `xinerama_head` and positioning

But Conky's Lua config doesn't support `dofile()` or includes natively in a clean way — it's a bit hacky.

**Realistically the best practice is just two files.** Since your only differences would be:
```
xinerama_head = 0 vs 1
gap_x = ...
gap_y = ...
```

The text block is identical. If you ever need to update the content, you update both — but that's a minor cost for a clean, simple setup.
