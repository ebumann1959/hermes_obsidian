## How Aaron's Obsidian + ChromaDB Layer Works

**What it does in one sentence:** Every Claude Code session you finish automatically gets converted into a browseable Obsidian note *and* embedded into a local ChromaDB index — zero manual curation, full semantic search of everything you've ever built with Claude.

### Architecture

```
~/.claude/projects/<project-slug>/<session-uuid>.jsonl  (written by Claude Code)
    │
    ▼  convert_[conversations.py](http://conversations.py)  (runs in Stop hook, ~2–5 s per session)
    │
~/.claude/vault/Conversations/<ProjectName>/<YYYY-MM-DD>__<uuid8>.md
    │
    ▼  seed_[chroma.py](http://chroma.py)  (runs nightly or via /resync-vault)
    │
~/.claude/chroma-db/  (ChromaDB PersistentClient, collection: project_context)
    ├── chroma-mcp  →  Claude Code MCP tools (semantic queries in any session)
    └── Smart Connections plugin  →  Obsidian mind-map / graph view
```

### Key Files

| Path | Purpose | Edit? |
|---|---|---|
| `~/.claude/settings.json` | Hooks + MCP server registration | Yes |
| `~/.claude/scripts/convert_[conversations.py](http://conversations.py)` | JSONL → Obsidian markdown | Yes (carefully) |
| `~/.claude/scripts/seed_[chroma.py](http://chroma.py)` | Markdown → ChromaDB embeddings | Yes (carefully) |
| `~/.claude/scripts/run_full_[sync.py](http://sync.py)` | Nightly wrapper (runs both scripts) | Yes |
| `~/.claude/vault/` | Obsidian vault root | Partial (don't touch `Conversations/`) |
| `~/.claude/chroma-db/` | ChromaDB storage | Never — wipe + reseed if corrupt |
| `~/.claude/logs/sync.log` | Audit trail for every sync | Read-only |

### Sync Contract (3 overlapping layers)

1. **Stop hook** — runs after every Claude session automatically. Calls `convert_[conversations.py](http://conversations.py)` (no flags). Takes 2–5 seconds. Converts only that session's JSONL to markdown. Does NOT seed Chroma (avoids write contention with the running MCP server).

2. **Nightly Windows Scheduled Task** (`Claude Vault Resync`, 03:00 daily) — runs `run_full_[sync.py](http://sync.py)` (incremental, mtime-based). Takes <1 minute on a warm index. Catches any sessions the hook missed, plus seeds all changed markdown into Chroma.

3. **Manual `/resync-vault` command** — runs `run_full_[sync.py](http://sync.py) --full` (re-embeds everything, ignores mtime). Takes 15–25 min on CPU. Use after editing project docs, adding new projects, or recovering from corruption.

### The `settings.json` Hook Structure

```json
{
  "hooks": {
    "Stop": [
      {
        "matcher": ".*",
        "hooks": [
          {
            "type": "command",
            "command": "python C:/Users/YourName/.claude/hooks/[no-secrets-check.py](http://no-secrets-check.py)"
          },
          {
            "type": "command",
            "command": "python C:/Users/YourName/.claude/scripts/convert_[conversations.py](http://conversations.py)"
          }
        ]
      }
    ]
  },
  "mcpServers": {
    "chroma": {
      "enable_tool_search": true
    }
  }
}
```

### Safety Rules (Hard-Won Lessons)

- **Never run `seed_[chroma.py](http://chroma.py)` while more than 1 `chroma-mcp.exe` is running** — concurrent writers corrupt the HNSW index silently. The script enforces this via preflight.
- **Never call `collection.delete(where={...})` on large collections** — the Chroma Rust bindings segfault above ~10k chunks. Always delete by explicit ID list instead.
- **Never call `collection.get(offset=N, where={...})`** — pagination + metadata filter is broken in current Chroma versions. Paginate without a where-filter.
- **Don't edit files in `vault/Conversations/`** — they're regenerated on every sync and your edits will be overwritten.
- **Kill by exact process name `chroma-mcp`** — not `*chroma*`. Razer's RGB software also uses "Chroma" in its process names.
- **If the index is corrupt, wipe and reseed from scratch** — it's safe because markdown is the source of truth: `rm -rf ~/.claude/chroma-db && python seed_[chroma.py](http://chroma.py) --full`.

### Prerequisites

- Python 3.11+
- `pip install chromadb sentence-transformers` (embedding model downloads on first run)
- `chroma-mcp` — either `pip install chroma-mcp` or download from the [chroma-mcp releases page](https://github.com/chroma-core/chroma-mcp)
- Obsidian desktop app (free)
- Claude Code CLI (`npm install -g @anthropic-ai/claude-code`)
