**What (1) changes:**

Right now, when you tell the system to reboot:
- systemd asks every service and user process to stop politely (SIGTERM).
- For each one that doesn't exit, it waits **90 seconds** before forcing it (SIGKILL).
- If multiple things are stuck, those 90s waits stack up, and you see the "frozen desktop, can't interact, won't reboot" state.

The drop-in files lower that grace period from **90s → 15s**, in two places:
- `/etc/systemd/system.conf.d/timeout.conf` — applies to system services (network, display manager, etc.)
- `/etc/systemd/user.conf.d/timeout.conf` — applies to *your* desktop session's services (kwin, plasma bits, pipewire, etc.) — this is usually where VR/desktop hangs actually live.

**What it does NOT do:**
- Doesn't touch the kernel, doesn't change drivers, doesn't affect VR.
- Doesn't affect normal running — only kicks in during shutdown/reboot/logout.
- Doesn't fix the root cause (whatever process is hanging), just stops it from holding the whole system hostage.

**Trade-offs:**
- A well-behaved service that legitimately needs 20–30s to flush state (databases, big file syncs) would get killed instead of shutting down cleanly. On a desktop workstation this is almost never a real problem — and 15s is still generous. If you're paranoid, we can set it to 30s instead.
