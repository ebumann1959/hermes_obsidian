## What you actually want — and the redesign

> One chat = one persistent hermes instance. Delete the chat = kill the process.

```
PWA  ──WebSocket──►  FastAPI server  ──stdin/stdout──►  hermes chat (process per thread)
                          │
                          ├─ thread_id → process map (in-memory)
                          ├─ idle eviction (kill after N min, restart on demand)

```

**Wins:**
- First message: ~30s (cold start, vault sync)

what is a pwa

## Assistant

**PWA = Progressive Web App.**

It's just a web page that does a few extra tricks so it behaves like an installed app:

- **Installable to your home screen** — when you tapped Safari's "Add to Home Screen" earlier, that's PWA behaviour. It got an icon, its own launch tile, and opens in standalone mode (no browser address bar / tabs).
- **Manifest file** (`manifest.json`) — tells iOS the app's name, colors, icon, display mode. We serve one from the Pi.
- **Service worker** (optional, we don't use one) — lets a PWA work offline by caching files. Yours is online-only since it talks live to the Pi.

So the "Hermes" thing on your iPhone home screen is just a webpage that's been dressed up to look and launch like a native app. No App Store, no Apple developer account, no Swift code — and you can change it just by editing the HTML on the Pi.

Most modern web apps you "install" (Twitter on the web, Spotify Web, etc.) are PWAs. It's the lightest way to ship a native-feeling app.
