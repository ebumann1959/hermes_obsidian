### `text_display` entities — the core primitive

These are vanilla entities (no mod needed) that render floating styled text in the world. Key properties:

- **`billboard: "center"`** — always rotates to face the viewer, just like your Unity DialogueHUD billboard
- **`background`** — ARGB int, so you can do a semi-transparent dark panel behind the text (speech bubble bg)
- **JSON text component** — full color, bold, italic, per-character styling
- **`view_range`** — how far it's visible
- **Multi-line** via `\n` in the text component
- Spawnable and killable via commands, so no plugin required if the bot has op

A speech bubble summon looks roughly like:
```
/summon minecraft:text_display 0 70 0 {
  text:'{"text":"I can\'t believe Richard did that.","color":"red"}',
  billboard:"center",
  background:1090519039,
  transformation:{scale:[1.0,1.0,1.0],...}
}
```

Susan speaks in red, Richard in blue, Gigi in pink — maps directly to your existing color assignments.

### Speech bubble lifecycle

On `dialogue_beat` from Pi:
1. Identify the speaker bot
2. `bot.chat('/summon minecraft:text_display ...')` above the bot's current position (Y+2.5)
3. Capture the entity UUID from the summon response (or tag it with a unique `CustomName`)
4. Set a 6-second timeout → `/kill @e[tag=sr_bubble_susan]`
5. Meanwhile, pathfinder can still move the bot — the text_display won't follow unless you attach it to the bot

**The "follows the character" problem** is the one real design decision: text_display entities don't parent to bots automatically. Options:
- **Static**: spawn bubble at current position, it stays there — fine if characters don't move mid-line
- **Riding**: `/ride <text_display_uuid> mount <bot_uuid>` — makes the display entity ride the bot, moves with it. This actually works in vanilla 1.20.2+.
- **Server-side update loop**: re-teleport the text_display every tick — noisy but works

The riding approach is cleanest. Bubble mounts the bot, moves with it, dismounts and despawns when the line ends.

A thin Paper plugin (~150 lines Java/Kotlin) gives you:
- HTTP or WebSocket endpoint the Node.js client calls: `POST /bubble {character, text, duration}`
- Plugin spawns a styled text_display, mounts it to the bot entity, schedules despawn
- Lets you use Paper's `Bukkit.getScheduler()` for clean tick-aligned updates
- Can smooth-interpolate bubble position if the bot moves
- Can add a second `block_display` entity as an actual speech bubble tail/pointer

Not required for v1 but makes the bubbles feel polished.
